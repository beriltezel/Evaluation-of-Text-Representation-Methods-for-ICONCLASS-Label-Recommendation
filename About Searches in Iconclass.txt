Iconclass is a classification system consisting of a hierarchical tree-like structure. It has ten main categories, and each category is identified by a number between 0 and 9. Under each main category there exists a multi-level hierarchy of subcategories. By adding a number or letter to the first number of the main category, the deeper levels of the categorization system are denoted. Iconclass notations are constructed in this way. Each notation also has a label connected to it, which is the textual representation of the category. For example, 2 represents the main category, Nature. If the number 3 is added to this, forming 23, this notation represents the subcategory time. If the letter R is added further to the classification code, 23R identifies the deeper subcategory day and night.
When a search is conducted in Iconclass, the labels are browsed and according to the search method, results are listed. The results consist of the notation and its label. Many searching algorithms either rely on the string match or the semantic similarity of the searched word or phrase and the results found in the source. This means, that the requested information is based on the textual data found in Iconclass. Altough Iconlcass is described as a classification system for the content of images, the search conducted is not between the images themselves, but between the conceptual descriptions assigned to the images. The example images of the artwork, which can be found under the labels when browsed in the Iconclass Website are mere representations of these labels. 
It is also important to note that one image can be classified under more than one notation, since there can be more than one meaning or concept extracted from that image. The assignment of the images to the labels are made by human cataloguers. Although this assignment conceptually sounds similar to the search conducted in Iconclass, it is something different from what is examined in this case.
